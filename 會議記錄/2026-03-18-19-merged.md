## 二、系統工程方法與流程

> **⚠️ 老師強調：** 本課程重點在於學習**系統工程方法**，而非單純完成技術產品。

課程將依循：

- **V-Model 流程**：需求定義 → 設計 → 測試 → 驗證
- **角色扮演機制**：每次會議從不同角色視角（需求方、設計方、測試方、使用者、投資人等）檢視系統

> **⚠️ 每次會議結構需包含：**
> 1. 上次議題 recap
> 2. 本次角色設定
> 3. 本次討論議題
> 4. 下次行動項目

## 五、✅ 最終決定事項

> **✅ 專題題目：智慧財務預警 Agent 系統**
>
> 幫助使用者避免超支、養成良好消費習慣，結合 LineBot 推播預警。

| 項目 | 決定 |
|------|------|
| 專題題目 | 智慧財務預警 Agent 系統 |
| AI 平台 | Claude Sonnet 4.6 (Anthropic) |
| 資料來源 | 使用者手動輸入 |
| 預警機制 | 雙層：Layer 1 Budget 上限 + Layer 2 ML 預測（Prophet） |
| ML 預測模型 | Prophet（原系統為 CNN + GRU，本專題資料量偏少故改用） |
| 冷啟動條件 | ≥ 7 天 且 ≥ 30 筆資料才啟動 ML |

### 系統 CONOPS（概念運作情境）

| 要素 | 內容 |
|------|------|
| Mission Objective | 幫助使用者避免超支、養成良好消費習慣 |
| Stakeholders | 使用者、Line 平台、金融資料來源 |
| CONOPS | 使用者連結 LineBot → 輸入消費記錄 → Agent 分析 → 預警推播 |
| Risks | 資料隱私、通知疲勞、資料準確性 |
